## 基本ルール
- 重要な判断は必ずAkiに確認してから実行
- 日本語をメインで使用（英語も対応可）
- セキュリティ最優先
- secrets/フォルダの情報は外部送信禁止
- ファイル削除前は必ずAkiに確認
- クライアント情報は該当プロジェクトフォルダ外に出さない
- 提案は優先度順に提示
- 不明点は勝手に進めず必ず確認する
- 作業終了時は必ずgit add . && git commit && git pushを実行する
- コミットメッセージは作業内容を日本語で簡潔に記載する

## セキュリティルール
- APIキー・パスワードはハードコーディング禁止
- secrets/フォルダはローカルのみ・GitHub push禁止
- 納品物は必ずセキュリティレビューを実施
- クライアントAの情報をクライアントBに使用禁止

## コミュニケーションスタイル
- 要点を簡潔にまとめて報告
- 複数の選択肢がある場合は優先度順に提示
- 長文より箇条書きを優先
- 日本語・英語どちらでも対応

## 作業開始時のルール
- 作業開始時は必ずgit pullで最新版を取得する
- secretary/todo.mdを確認してから作業を始める

## エラー・問題発生時
- エラーが発生したら必ずAkiに報告してから修正する
- 自己判断で重要なファイルを変更しない
- 解決できない問題は原因と選択肢を提示して確認を取る

## ファイル管理ルール
- 新しいファイルを作成したらgitignoreの対象か確認する
- secretsフォルダのファイルは絶対にgit addしない
- 大きな変更前は必ずバックアップを取る

## プロジェクト管理ルール
- 新しいプロジェクトはprojects/フォルダ内に作成する
- 各プロジェクトにREADME.mdを必ず作成する
- クライアントプロジェクトはclient-[会社名]の形式で命名する
- 完了したタスクはtodo.mdから削除せずdone.mdに移動する

## Claude Codeの動作ルール
- 一度に大量のファイルを変更しない（5ファイル以内）
- 長い処理の前に必ず内容をAkiに確認する
- 提案が複数ある場合は番号付きで選択肢を提示する
- 不明な指示は勝手に解釈せず必ず確認する

## 多拠点運用ルール
- 家PC・会社PC・福岡PCで同じ環境を維持する
- PC固有の設定はローカルのみに保存する
- 環境依存のパスはハードコーディングしない

## メンテナンスルール
- 週1回todo.mdを見直して完了タスクをdone.mdに移動する
- 月1回secrets/フォルダの内容を棚卸しする
